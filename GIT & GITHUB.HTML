Git Basics & Setup

Estimated Time: ~3 Hours  |  Configuration, Init, Staging, First Commits

⚙️
# Git Installation & Configuration
Initial setup — identity, editor, aliases
SETUP ▼

Before using Git, configure your identity. These values are embedded in every commit you make.

## Check Git Version
bash
git --version
## Set Global Identity
bash
git config --global user.name "Your Name"
git config --global user.email "you@example.com"
# Project-level (no --global)
git config user.name "Project Name"
## Default Branch & Editor
bash
git config --global init.defaultBranch main
git config --global core.editor "code --wait"   # VS Code
git config --global core.editor "vim"           # Vim
## View & Edit Config
bash
git config --list               # All settings
git config --global --list       # Global only
git config --global --edit       # Open .gitconfig in editor
git config user.name             # Get single value
## Useful Aliases
bash
git config --global alias.st status
git config --global alias.co checkout
git config --global alias.br branch
git config --global alias.lg "log --oneline --graph --all"
# Usage:
git st     # same as git status
git lg     # pretty log graph

# Initialize & Clone Repositories
git init, git clone
CORE ▼
## git init
bash
git init                   # Init in current folder
git init my-project        # Create + init new folder
git init --bare repo.git   # Bare repo (server-side)
## git clone
bash
git clone https://github.com/user/repo.git
git clone git@github.com:user/repo.git          # SSH
git clone URL my-folder                           # Custom folder name
git clone --depth 1 URL                           # Shallow clone (fastest)
git clone --branch develop URL                   # Clone specific branch
git clone --recurse-submodules URL              # With submodules

# Staging Area (Index)
git add, git rm, git mv, git status
CORE ▼
## git status
bash
git status            # Full output
git status -s         # Short/compact format
git status -sb        # Short + branch info
## git add
bash
git add file.txt       # Stage one file
git add src/           # Stage entire folder
git add .              # Stage all changes
git add -A             # Stage all (including deletions)
git add -p             # Interactive patch staging (hunk by hunk)
git add -u             # Stage modified + deleted (not untracked)
git add *.js           # Stage all .js files
## git rm & git mv
bash
git rm file.txt              # Remove file & stage deletion
git rm --cached file.txt    # Untrack but keep on disk
git rm -r folder/           # Remove entire folder
git mv old.txt new.txt      # Rename/move file

bash

# After adding to .gitignore, remove cached:
git rm -r --cached .
git add .

Commits & Viewing History

Estimated Time: ~3 Hours  |  Commit, Diff, Log, Stash, Tags

💾
# Committing Changes
git commit, amend, messages
COMMITS ▼
## Basic Commit
bash
git commit -m "feat: add login page"
git commit -am "fix: update styles"   # Stage tracked files + commit
git commit                              # Open editor for message
git commit --allow-empty -m "chore: trigger CI"
## Amend Last Commit
bash
git commit --amend -m "corrected message"
git commit --amend --no-edit   # Keep message, add staged changes
⚠️ Never amend commits that have been pushed to a shared branch — it rewrites history.
## Conventional Commit Format
format
feat:     new feature
fix:      bug fix
docs:     documentation
style:    formatting (no logic change)
refactor: code restructure
test:     add/update tests
chore:    maintenance tasks
perf:     performance improvement

# Example:
git commit -m "feat(auth): add JWT refresh token support"
🔍
# Viewing Differences (git diff)
Unstaged, staged, between commits
DIFF ▼
bash
git diff                        # Unstaged changes
git diff --staged               # Staged vs last commit
git diff HEAD                    # All changes vs last commit
git diff main..feature          # Between branches
git diff abc123 def456          # Between two commits
git diff --stat                  # Summary of changes
git diff --name-only             # Only filenames
git diff HEAD~3..HEAD           # Last 3 commits
📜
# Viewing Commit History (git log)
log, shortlog, show
HISTORY ▼
bash
git log                                      # Full log
git log --oneline                            # One line per commit
git log --oneline --graph --all             # Visual branch graph
git log --oneline -10                       # Last 10 commits
git log --author="Name"                    # Filter by author
git log --since="2024-01-01"              # After date
git log --grep="fix"                       # Search commit messages
git log -S"function name"                  # Search added/removed code
git log --follow file.txt                  # History including renames
git log --format="%H | %an | %s"          # Custom format
git log --no-merges                         # Exclude merge commits
git show abc123                              # Show a specific commit
git shortlog -sn                             # Commits per author
git blame file.txt                          # Who changed each line
📦
# Stashing Changes
Temporarily save uncommitted work
STASH ▼
bash
git stash                              # Stash tracked changes
git stash push -m "wip: feature x"   # Named stash
git stash -u                          # Include untracked files
git stash -a                          # Include ignored files too
git stash list                        # Show all stashes
git stash pop                         # Apply + remove latest stash
git stash apply stash@{2}           # Apply specific stash
git stash drop stash@{0}            # Delete specific stash
git stash clear                      # Delete all stashes
git stash show -p                    # Show stash diff
git stash branch new-branch        # Create branch from stash
🏷️
# Tags
Lightweight and annotated tags, releases
TAGS ▼
bash
git tag                             # List all tags
git tag v1.0.0                     # Lightweight tag
git tag -a v1.0.0 -m "Release 1.0" # Annotated tag
git tag -a v1.0.0 abc123         # Tag a specific commit
git show v1.0.0                    # Show tag details
git push origin v1.0.0            # Push one tag
git push origin --tags            # Push all tags
git tag -d v1.0.0                 # Delete local tag
git push origin --delete v1.0.0  # Delete remote tag
git checkout v1.0.0               # Checkout a tag (detached HEAD)

Branching & Merging

Estimated Time: ~4 Hours  |  Branches, Merge, Rebase, Cherry-pick, Conflicts

🌿
# Branch Management
Create, switch, rename, delete branches
BRANCH ▼
## Create & Switch
bash
git branch                         # List local branches
git branch -a                      # List all (local + remote)
git branch -r                      # Remote branches only
git branch feature/login          # Create branch
git checkout feature/login        # Switch to branch
git checkout -b feature/login    # Create + switch
git switch feature/login          # Modern switch (Git 2.23+)
git switch -c feature/login      # Create + switch (modern)
git switch -                      # Switch to previous branch
## Rename & Delete
bash
git branch -m old-name new-name        # Rename branch
git branch -d feature/login            # Safe delete (merged)
git branch -D feature/login            # Force delete
git push origin --delete feature/login # Delete remote branch
git branch --merged                    # Branches merged into current
git branch --no-merged                 # Unmerged branches
🔀
# Merging
Merge strategies, fast-forward, squash
MERGE ▼
bash
git merge feature/login            # Merge into current branch
git merge --no-ff feature/login   # Force merge commit (no fast-forward)
git merge --ff-only feature       # Only if fast-forward possible
git merge --squash feature        # Squash all into one commit
git merge --abort                  # Cancel merge in progress
git merge --continue               # Continue after resolving conflicts
## Conflict Resolution
bash
# 1. After a conflict, files look like:
<<<<<<< HEAD
your changes
=======
their changes
>>>>>>> feature/login

# 2. Edit the file, then:
git add conflicted-file.txt
git commit

# Useful conflict tools:
git mergetool               # Launch merge tool (vimdiff, kdiff3...)
git checkout --ours file   # Accept our version
git checkout --theirs file # Accept their version
🔄
# Rebase
Linear history, interactive rebase
REBASE ▼
bash
git rebase main                   # Rebase current branch onto main
git rebase -i HEAD~3             # Interactive rebase last 3 commits
git rebase --onto main A B      # Rebase branch B onto main, starting after A
git rebase --abort               # Cancel rebase
git rebase --continue            # Continue after conflict resolution
git rebase --skip                # Skip current patch
## Interactive Rebase Commands
interactive rebase options
pick    # use commit as-is
reword  # use commit, edit message
edit    # pause to amend
squash  # meld into previous commit
fixup   # like squash, discard message
drop    # remove commit entirely
exec    # run shell command after commit
🍒
# Cherry-Pick
Apply specific commits from other branches
CHERRY ▼
bash
git cherry-pick abc123              # Apply one commit
git cherry-pick abc123 def456       # Multiple commits
git cherry-pick abc123..def456     # Range of commits
git cherry-pick -n abc123          # Apply but don't commit
git cherry-pick --abort             # Cancel
git cherry-pick --continue          # Continue after conflict

bash

# SSH Key Setup for GitHub
Generate, add, test SSH connection
SSH ▼
bash
# Generate SSH key
ssh-keygen -t ed25519 -C "you@example.com"

# Start SSH agent & add key
eval "$(ssh-agent -s)"
ssh-add ~/.ssh/id_ed25519

# Copy public key to clipboard
cat ~/.ssh/id_ed25519.pub          # Paste this in GitHub SSH settings

# Test connection
ssh -T git@github.com

# Change existing repo from HTTPS to SSH
git remote set-url origin git@github.com:user/repo.git

Undoing Changes & Reset

Estimated Time: ~3 Hours  |  Restore, Revert, Reset, Reflog, Clean

↩️
# Undoing Working Tree Changes
restore, checkout, clean
UNDO ▼
bash
# Discard changes in working directory
git restore file.txt             # Discard unstaged changes (modern)
git restore .                     # Discard all unstaged changes
git checkout -- file.txt         # Same (older syntax)

# Unstage staged files
git restore --staged file.txt   # Unstage specific file
git restore --staged .           # Unstage everything
git reset HEAD file.txt          # Older unstage method

# Restore to a specific commit
git restore --source=HEAD~2 file.txt

# Remove untracked files
git clean -n         # Dry run (show what will be removed)
git clean -f         # Remove untracked files
git clean -fd        # Remove untracked files + directories
git clean -fX        # Remove only ignored files
git clean -fdx       # Remove all untracked + ignored
🔁
# git reset
soft, mixed, hard — move HEAD
RESET ▼
bash
# --soft: move HEAD, keep stage + working tree
git reset --soft HEAD~1    # Undo last commit, keep changes staged

# --mixed (default): move HEAD, unstage, keep working tree
git reset HEAD~1           # Undo commit, unstage changes
git reset abc123           # Reset to specific commit

# --hard: move HEAD, discard staged + working tree
git reset --hard HEAD~1    # Undo commit + discard all changes
git reset --hard origin/main # Match remote branch
⚠️ --hard is destructive. Use git reflog to recover if needed.
🔙
# git revert
Safe undo — creates a new commit
REVERT ▼
bash
git revert HEAD                 # Revert last commit
git revert abc123               # Revert specific commit
git revert HEAD~3..HEAD        # Revert a range
git revert -n abc123           # Revert without auto-committing
git revert --abort              # Cancel revert in progress
🗂️
# Reflog — Your Safety Net
Recover lost commits and branches
REFLOG ▼
bash
git reflog                        # Show HEAD movement history
git reflog show feature-branch  # Reflog for specific branch

# Recover deleted branch
git reflog                        # Find the commit hash
git branch recovered-branch abc123

# Recover from bad reset --hard
git reflog                        # Find the right HEAD@{n}
git reset --hard HEAD@{3}

# Pull Request Workflow
Fork, branch, PR, review, merge
PR ▼
workflow
# 1. Fork repo on GitHub, then clone your fork
git clone git@github.com:you/repo.git

# 2. Add upstream remote
git remote add upstream git@github.com:original/repo.git

# 3. Create feature branch
git switch -c feature/my-change

# 4. Make changes, commit
git add .
git commit -m "feat: my change"

# 5. Sync with upstream before push
git fetch upstream
git rebase upstream/main

# 6. Push to your fork
git push -u origin feature/my-change

# 7. Open Pull Request on GitHub / gh pr create

# GitHub Actions (CI/CD)
Workflow YAML structure & key commands
ACTIONS ▼
.github/workflows/ci.yml
name: CI
on:
  push:
    branches: [main, develop]
  pull_request:
    branches: [main]

jobs:
  build:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - name: Setup Node.js
        uses: actions/setup-node@v4
        with:
          node-version: '20'
      - name: Install dependencies
        run: npm ci
      - name: Run tests
        run: npm test
      - name: Build
        run: npm run build

Estimated Time: ~4 Hours  |  Bisect, Worktree, Submodules, Hooks, Signing, Large Files

🔬
# git bisect — Find the Bug Commit
Binary search through history
BISECT ▼
bash
git bisect start
git bisect bad                     # Current commit has bug
git bisect good v1.0.0            # This tag was working
# Git checks out midpoint — test and mark:
git bisect good                     # This commit is OK
git bisect bad                      # This commit has the bug
# Git narrows down... repeats until found
git bisect reset                    # Back to original branch

# Automated bisect with a test script
git bisect run npm test            # 0=good, non-zero=bad
🌲
# git worktree
Multiple branches checked out simultaneously
WORKTREE ▼
bash
git worktree add ../hotfix hotfix/v1   # Checkout branch in new folder
git worktree add -b fix/bug ../fix     # Create new branch in new folder
git worktree list                       # List all worktrees
git worktree remove ../hotfix          # Remove worktree
git worktree prune                      # Clean stale worktrees
🧩
# Submodules
Embed repos inside repos
SUBMODULE ▼
bash
git submodule add URL path/to/sub    # Add submodule
git submodule init                     # Initialize submodules
git submodule update                   # Checkout pinned commits
git submodule update --init --recursive # All nested submodules
git submodule update --remote         # Update to latest remote
git submodule foreach 'git pull'      # Run command in each submodule
git submodule status                   # Show submodule state
git clone --recurse-submodules URL   # Clone with submodules

# Signing Commits (GPG)
Verified commits on GitHub
GPG ▼
bash
# Generate GPG key
gpg --full-generate-key

# Get key ID
gpg --list-secret-keys --keyid-format=long

# Configure Git to use GPG key
git config --global user.signingkey YOUR_KEY_ID
git config --global commit.gpgsign true

# Sign individual commit
git commit -S -m "signed commit"

# Verify a commit signature
git verify-commit abc123

# Export public key for GitHub
gpg --armor --export YOUR_KEY_ID

# Git LFS (Large File Storage)
Track binaries, images, models outside repo
LFS ▼
